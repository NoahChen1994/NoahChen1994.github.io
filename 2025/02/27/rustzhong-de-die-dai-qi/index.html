Rust中的迭代器

1. 迭代器概述

Rust 中的迭代器是一个实现了 Iterator trait 的对象。迭代器通过 next 方法返回集合的元素，并且每次迭代返回的是一个 Option 类型的值（即 Some(item) 或 None）。next 方法会持续调用直到所有元素都被返回，最终返回 None。

在 Rust 中，迭代器的特点是惰性求值，意味着它们不会在创建时立即开始计算，而是等到实际需要时才开始工作。这种惰性求值模式使得 Rust 的迭代器具有高效性和灵活性。

2. 使用迭代器

2.1 创建和使用迭代器

Rust 提供了许多集合类型（如 Vec、HashMap 等），它们都实现了 IntoIterator trait，允许你通过 .iter() 或 .into_iter() 方法生成迭代器。

let v = vec![1, 2, 3];

for x in v.iter() {
    println!("{}", x);
}

在上面的代码中，v.iter() 返回一个不可变引用迭代器。for 循环会在每次迭代时自动调用 next() 方法，直到集合中的元素被遍历完。

你还可以使用 .into_iter() 生成一个所有权转移的迭代器：

let v = vec![1, 2, 3];

for x in v.into_iter() {
    println!("{}", x);
}

into_iter() 会消耗 v，并将所有权转移到迭代器中。

2.2 迭代器的惰性

Rust 中的迭代器方法是惰性求值的，这意味着它们不会立即计算，而是等到你需要结果时才会执行。这有助于提高性能，因为你可以链式调用多个迭代器方法，Rust 只会在需要时才执行这些操作。

例如：

let v = vec![1, 2, 3, 4, 5];
let result: Vec<_> = v.iter()
    .map(|x| x * 2)  // 将每个元素乘以 2
    .filter(|x| x > &5)  // 只保留大于 5 的元素
    .collect();  // 将结果收集到一个新的 Vec 中

println!("{:?}", result);  // 输出: [6, 8, 10]

在这个例子中，map 和 filter 是惰性操作，它们不会立即对 v 进行修改，直到调用 .collect() 时才会执行计算。

3. 迭代器方法

Rust 提供了许多迭代器方法，可以帮助开发者轻松地对集合进行操作。常见的迭代器方法包括：

  • map: 对集合中的每个元素执行某种转换操作。
  • filter: 筛选符合条件的元素。
  • fold: 将集合中的所有元素归约成一个单一的值。
  • any: 检查集合中是否有任何元素满足某个条件。
  • all: 检查集合中是否所有元素都满足某个条件。
  • sum: 对集合中的所有元素进行求和操作。

例如，使用 map 和 filter 方法：

let v = vec![1, 2, 3, 4, 5];
let result: Vec<i32> = v.iter()
    .map(|x| x * 2)  // 每个元素乘以 2
    .filter(|x| x > &5)  // 只保留大于 5 的元素
    .collect();  // 将结果收集到一个新的 Vec 中
println!("{:?}", result);  // 输出: [6, 8, 10]

fold 方法示例：

let sum: i32 = (1..=5).fold(0, |acc, x| acc + x);
println!("Sum: {}", sum);  // 输出: Sum: 15

fold 方法用于将集合中的所有元素合并为一个值。在上面的例子中，fold 将集合中的元素进行求和，初始值是 0，并将结果返回。

4. 自定义迭代器

Rust 允许你定义自己的迭代器类型，方法是实现 Iterator trait。Iterator trait 定义了两个核心方法：

  • next(): 获取迭代器中的下一个元素。
  • size_hint(): 给出估计的剩余元素数量。

例如，定义一个简单的迭代器：

struct Counter {
    count: u32,
}

impl Counter {
    fn new() -> Counter {
        Counter { count: 0 }
    }
}

impl Iterator for Counter {
    type Item = u32;

    fn next(&mut self) -> Option<Self::Item> {
        if self.count < 5 {
            self.count += 1;
            Some(self.count)
        } else {
            None
        }
    }
}

fn main() {
    let counter = Counter::new();

    for num in counter {
        println!("{}", num);
    }
}

这个迭代器会返回从 1 到 5 的数字。

5. 迭代器的性能

Rust 的迭代器方法使用惰性求值，因此能够减少不必要的计算。当你链接多个迭代器时，它们不会立即执行，而是将所有操作推迟到你需要结果时。这样可以减少中间计算的开销。

let v = vec![1, 2, 3, 4, 5];
let result: Vec<_> = v.iter()
    .map(|x| x * 2)
    .filter(|x| x > &5)
    .collect();

在上面的代码中，map 和 filter 仅在调用 .collect() 时才会执行，而不会立即处理每个元素。

6. 迭代器与所有权

Rust 的迭代器可以通过 move 语义传递所有权，这意味着迭代器可以在不复制数据的情况下处理值。into_iter() 会消耗容器并返回每个元素的所有权，而 .iter() 只是返回元素的不可变引用。

例如，使用 into_iter 获取所有权：

let v = vec![1, 2, 3];

for x in v.into_iter() {
    println!("{}", x);
}

在此例中，v.into_iter() 会消耗 v，并将其所有权转移到迭代器中。

性能对比：循环 VS 迭代器

为了决定使用哪个实现，我们需要知道哪个版本的 search 函数更快一些：是直接使用 for 循环的版本还是使用迭代器的版本。

我们运行了一个性能测试，通过将阿瑟·柯南·道尔的“福尔摩斯探案集”的全部内容加载进 String 并寻找其中的单词 “the”。如下是 for 循环版本和迭代器版本的 search 函数的性能测试结果：

test bench_search_for  ... bench:  19,620,300 ns/iter (+/- 915,700)
test bench_search_iter ... bench:  19,234,900 ns/iter (+/- 657,200)

结果迭代器版本还要稍微快一点！这里我们将不会查看性能测试的代码，我们的目的并不是为了证明他们是完全等同的，而是得出一个怎样比较这两种实现方式性能的基本思路。

对于一个更全面的性能测试，将会检查不同长度的文本、不同的搜索单词、不同长度的单词和所有其他的可变情况。这里所要表达的是：迭代器，作为一个高级的抽象，被编译成了与手写的底层代码大体一致性能代码。迭代器是 Rust 的 零成本抽象（zero-cost abstractions）之一，它意味着抽象并不会引入运行时开销，它与本贾尼·斯特劳斯特卢普（C++ 的设计和实现者）在 “Foundations of C++”（2012） 中所定义的 零开销（zero-overhead）如出一辙：

In general, C++ implementations obey the zero-overhead principle: What you don’t use, you don’t pay for. And further: What you do use, you couldn’t hand code any better.

  • Bjarne Stroustrup “Foundations of C++”

从整体来说，C++ 的实现遵循了零开销原则：你不需要的，无需为他们买单。更有甚者的是：你需要的时候，也不可能找到其他更好的代码了。

  • 本贾尼·斯特劳斯特卢普 “Foundations of C++”

7. 总结

Rust 的迭代器提供了一个强大且灵活的方式来操作集合。通过实现 Iterator trait，Rust 提供了丰富的迭代器方法，如 map、filter、fold 等，帮助开发者高效地处理数据。Rust 中的迭代器是惰性求值的，这使得它们非常高效，可以将多个操作链式组合，而不会立即执行。
