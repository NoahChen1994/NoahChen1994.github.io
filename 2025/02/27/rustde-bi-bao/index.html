Rust中的闭包

Awsome

更新日期:   2025-02-27
文章字数:   2.4k
阅读时长:   9 分

Rust 中的闭包(Closures)是匿名函数，允许在函数内定义并传递行为。闭包是一种功能强大的工具，允许你在执行时捕获周围环境中的变量，并且能够传递给其他函数进行使用。闭包与普通函数不同之处在于，闭包可以捕获其调用环境中的变量，并且能够按需要改变它们的值。

1. 闭包 (Closures) 的定义

闭包的基本形式如下：

let closure_name = |参数1, 参数2, ...| 表达式;
  • |参数1, 参数2, ...|：闭包的参数部分，类似于函数的参数。
  • 表达式：闭包体，可以是任何合法的 Rust 表达式。

例如：

let add = |a, b| a + b;
println!("The sum is: {}", add(2, 3));  // 输出：The sum is: 5

这是一个简单的闭包，接收两个参数 a 和 b，并返回它们的和。

2. 闭包的类型推断

Rust 的闭包具有类型推断能力，编译器会根据闭包的使用场景推断出闭包的类型。这意味着你通常不需要明确指定闭包的类型。

例如：

let multiply = |x, y| x * y;  // Rust 会自动推断类型
println!("{}", multiply(4, 5));  // 输出 20

闭包的参数和返回值类型通常会根据传递给闭包的参数自动推导。如果你需要，Rust 也允许显式指定类型。

3. 闭包捕获环境中的变量

闭包与常规函数的最大区别在于，它可以捕获并使用其所在环境中的变量。Rust 提供了三种不同的方式来捕获这些环境变量：按值、按引用和按可变引用。

  • 按值：闭包会取得变量的所有权。
  • 按引用：闭包会借用变量（不可变引用）。
  • 按可变引用：闭包会借用变量（可变引用）。

在 Rust 中，闭包捕获外部变量时，可以通过以下方式进行：

let x = 5;
let print_x = || println!("x is: {}", x);  // 闭包捕获 x 的引用
print_x();  // 输出: x is: 5

闭包 print_x 捕获了 x 的不可变引用，这样就能在闭包内访问它。

如果闭包需要修改外部变量，则可以通过可变引用捕获变量：

let mut x = 5;
let mut change_x = || {
    x += 1;  // 修改 x 的值
};
change_x();
println!("{}", x);  // 输出: 6

4. 闭包和函数的比较

闭包与普通函数有很多相似之处，但也有一些重要的不同之处。一个关键的区别是，闭包能够捕获并使用其环境中的变量，而普通函数则不能。下面通过例子进行比较：

普通函数：

fn add(a: i32, b: i32) -> i32 {
    a + b
}

闭包：

let add = |a, b| a + b;

闭包在灵活性上优于普通函数，因为它能够自由访问和修改外部环境中的变量。

5. 闭包类型

Rust 通过三种类型的闭包来实现对捕获环境变量的不同处理方式：

  • Fn：闭包按不可变引用捕获变量，不能修改捕获的变量。
  • FnMut：闭包按可变引用捕获变量，可以修改捕获的变量。
  • FnOnce：闭包按值捕获变量，并取得它们的所有权，闭包只能调用一次，因为它消耗了所有权。

这些类型用于 Rust 中的类型系统，以确保闭包在运行时的行为是安全的。

let x = 5;
let f: Fn() = || println!("x is: {}", x);  // 使用 Fn 类型
f();

闭包的具体类型取决于闭包如何捕获环境变量。例如，Fn 类型闭包只能以不可变引用的方式捕获环境变量，而 FnMut 则允许修改这些变量。

6. 闭包的应用

闭包是 Rust 中非常强大的工具，它们常用于迭代器、异步编程、回调等多种场景。以下是一些常见的应用场景：

  • 迭代器和闭包：Rust 中的许多标准库类型，如迭代器，都使用了闭包。
let numbers = vec![1, 2, 3];
let sum: i32 = numbers.iter().map(|x| x * 2).sum();  // 使用闭包进行处理
println!("{}", sum);  // 输出 12

在这里，map 方法接受一个闭包，执行对每个元素的处理（在此例中是将每个元素乘以 2）。

  • 回调函数：闭包经常被用于回调机制。例如，在异步编程或事件驱动的编程中，闭包作为回调函数非常常见。
fn execute<F>(f: F)
where
    F: Fn(),
{
    f();  // 执行回调函数
}

let closure = || println!("Callback executed!");
execute(closure);  // 调用回调

7. 示例：使用闭包缓存结果

我们将通过一个示例展示如何使用闭包和缓存机制来优化计算过程。在这个示例中，我们模拟了一个健身计划生成器，并使用闭包来缓存昂贵的计算结果，避免重复计算。

main.rs

use std::thread;
use std::time::Duration;

fn main() {
    let simulted_user_specifid_value = 10;  // 假设的用户输入的健身强度值
    let simulted_random_number = 7;  // 随机生成的数字，用于决策
    generate_workout(simulted_user_specifid_value, simulted_random_number);
}

// 模拟生成健身计划的算法
fn simulated_expensive_calculated(intensity: u32) -> u32 {
    println!("calculating slowly...");  // 打印信息，表示计算正在进行
    thread::sleep(Duration::from_secs(2));  // 模拟一个耗时的操作，延迟 2 秒
    intensity  // 返回输入的健身强度
}

fn generate_workout(intensity: u32, random_number: u32) {
    // 虽然可以使用变量存储函数结果，但是每次进入时都需要重新生成健身计划，非常耗时
    // let calculte_result = simulated_expensive_caculated(intensity);

    // 我们期望只有在需要计算时才调用算法，同时通过闭包缓存计算结果
    let mut calculated_closures = Cacher::new(|num| {
        println!("calculating slowly...");  // 闭包内部的计算模拟
        thread::sleep(Duration::from_secs(2));  // 模拟耗时计算
        num  // 返回输入值
    });

    // 根据健身强度生成不同的计划
    if intensity < 30 {
        println!("Today do {} pushups", calculated_closures.value(intensity)); 
        println!("Next, do {} situps", calculated_closures.value(intensity));  
    } else {
        if random_number == 3 {
            println!("Take a break today!")  // 如果随机数是 3，休息，这种情况下不需要去调用算法
        } else {
            println!("Today, run for {} minutes", calculated_closures.value(intensity)); 
        }
    }
}

// Cacher 结构体用于缓存昂贵的计算结果
// 它接收一个闭包作为参数，缓存该闭包的计算结果
struct Cacher<T> where T: Fn(u32) -> u32 {
    calculation: T,  // 闭包函数，用于计算
    value: Option<u32>,  // 缓存计算结果
}

impl<T> Cacher<T> where T: Fn(u32) -> u32 {
    fn new(calculation: T) -> Cacher<T> {
        Cacher {
            calculation,
            value: None, 
        }
    }

    fn value(&mut self, args: u32) -> u32 {
        match self.value {
            Some(v) => v,  // 如果缓存已有值，直接返回
            None => {
                let v = (self.calculation)(args);  // 执行闭包计算
                self.value = Some(v);  // 缓存结果
                v  // 返回计算结果
            }
        }
    }
}

simulated_expensive_calculated 函数：

这是一个模拟计算的函数，它会模拟一个需要较长时间才能完成的昂贵操作。在本示例中，我们使用 thread::sleep 来延迟 2 秒钟，表示计算的过程。

generate_workout 函数：

该函数根据用户输入的强度值和一个随机数生成健身计划。为了避免每次生成计划时都进行昂贵的计算，我们使用了 Cacher 结构体来缓存计算结果。

  • 闭包：我们创建了一个闭包 calculated_closures，它接受一个 u32 类型的参数并返回该值。闭包执行时会模拟计算过程，并将结果缓存起来。
  • 缓存机制：通过 Cacher，我们确保同一个输入值 intensity 在同一次调用中只会计算一次，从而提高效率。

Cacher 结构体：

Cacher 是一个泛型结构体，用于缓存昂贵计算的结果。它接受一个闭包 calculation，并缓存该闭包执行后的结果。

  • value 方法：该方法会检查是否已经有缓存结果。如果有，它会直接返回缓存的值；如果没有，它会执行闭包计算并将结果缓存。

闭包与缓存的结合：

闭包的关键特性是能够捕获并使用外部变量。在本示例中，Cacher 通过一个闭包 calculation 来延迟执行计算，并且只有第一次使用时才会执行计算，之后直接返回缓存值。这种优化在需要多次执行相同计算的场景中非常有用。

Cacher 的优势：

  • 懒惰计算(memoization 或 lazy evaluation（惰性求值）)：闭包和缓存的结合使得我们能在需要时才执行计算，而不是每次都进行昂贵的操作。
  • 性能优化：在调用 value 方法时，只有第一次调用会进行耗时的计算，后续调用将直接返回缓存结果，节省时间和资源。

总结

  • 闭包：闭包是 Rust 中的一种匿名函数，它可以捕获外部环境中的变量并与其共同工作。闭包比普通函数更具灵活性，可以捕获并操作外部变量。
  • 捕获方式：闭包可以通过值、不可变引用或可变引用捕获外部变量。
  • 闭包的类型：闭包有三种类型：Fn、FnMut 和 FnOnce，分别表示不同的捕获和调用行为。
  • 应用场景：闭包在 Rust 中的应用非常广泛，包括迭代器、回调函数、异步编程等。

闭包是 Rust 强大而灵活的功能之一，帮助我们编写更简洁、易于组合的代码。通过理解闭包如何捕获环境、如何进行类型推断以及如何使用它们，我们可以在 Rust 中更高效地编写程序。
