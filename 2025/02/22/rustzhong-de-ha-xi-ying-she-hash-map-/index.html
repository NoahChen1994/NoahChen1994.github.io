Rust中的哈希映射(Hash Map)

Awsome
Rust
发布日期:   2025-02-22
更新日期:   2025-02-22
文章字数:   1.2k
阅读时长:   5 分

哈希映射是一个存储键值对（key-value pairs）的集合。每个键是唯一的，通过键可以快速访问对应的值。Rust 提供了标准库中的 HashMap 类型来实现哈希映射。

1. 导入 std::collections::HashMap

在使用哈希映射时，我们需要从 std::collections 模块中导入 HashMap 类型：

use std::collections::HashMap;

2. 创建一个哈希映射

我们可以通过 HashMap::new() 来创建一个空的哈希映射：

use std::collections::HashMap;

fn main() {
    let mut scores = HashMap::new();  // 创建一个空的哈希映射
    scores.insert(String::from("Blue"), 10);  // 向哈希映射中插入键值对
    scores.insert(String::from("Yellow"), 50); // 插入第二个键值对

    println!("{:?}", scores);  // 打印哈希映射
}

3. 查找值

哈希映射提供了 .get() 方法来根据键查找值，返回的是 Option 类型，如果键存在返回 Some(value)，如果不存在返回 None。

use std::collections::HashMap;

fn main() {
    let mut scores = HashMap::new();
    scores.insert(String::from("Blue"), 10);
    scores.insert(String::from("Yellow"), 50);

    let blue_score = scores.get(&String::from("Blue"));
    match blue_score {
        Some(score) => println!("Blue team's score: {}", score),
        None => println!("No score for Blue team."),
    }
}

4. 更新哈希映射中的值

我们可以使用 .insert() 来插入新的键值对。如果键已经存在，insert 会替换旧值：

use std::collections::HashMap;

fn main() {
    let mut scores = HashMap::new();
    scores.insert(String::from("Blue"), 10);
    scores.insert(String::from("Yellow"), 50);

    // 更新值
    scores.insert(String::from("Blue"), 25);  // Blue队的得分更新为25
    println!("{:?}", scores);
}

如果你只想在键不存在时插入值，可以使用 .entry() 方法配合 .or_insert()：

use std::collections::HashMap;

fn main() {
    let mut scores = HashMap::new();
    scores.insert(String::from("Blue"), 10);
    scores.insert(String::from("Yellow"), 50);

    // 只有键不存在时才插入
    scores.entry(String::from("Blue")).or_insert(100);
    scores.entry(String::from("Green")).or_insert(60);

    println!("{:?}", scores);  // 输出: {"Blue": 10, "Yellow": 50, "Green": 60}
}

5. 移除元素

使用 .remove() 方法可以通过键删除哈希映射中的元素：

use std::collections::HashMap;

fn main() {
    let mut scores = HashMap::new();
    scores.insert(String::from("Blue"), 10);
    scores.insert(String::from("Yellow"), 50);

    scores.remove("Blue");  // 移除 "Blue" 键

    println!("{:?}", scores);  // 输出: {"Yellow": 50}
}

6. 遍历哈希映射

我们可以使用 for 循环来遍历哈希映射中的所有键值对：

use std::collections::HashMap;

fn main() {
    let mut scores = HashMap::new();
    scores.insert(String::from("Blue"), 10);
    scores.insert(String::from("Yellow"), 50);

    // 遍历哈希映射
    for (key, value) in &scores {
        println!("{}: {}", key, value);
    }
}

7. 使用哈希映射存储复合类型

哈希映射不仅可以存储基础数据类型，还可以存储结构体、枚举等复杂数据类型。例如，我们可以将结构体作为哈希映射中的值。

use std::collections::HashMap;

#[derive(Debug)]
struct Player {
    name: String,
    score: i32,
}

fn main() {
    let mut players = HashMap::new();

    // 创建并插入结构体
    players.insert(String::from("Player1"), Player {
        name: String::from("Alice"),
        score: 100,
    });

    players.insert(String::from("Player2"), Player {
        name: String::from("Bob"),
        score: 200,
    });

    // 打印玩家信息
    for (key, player) in &players {
        println!("{}: {:?}", key, player);
    }
}

8. 注意事项：键的所有权

在将数据插入到哈希映射时，键会被转移到哈希映射中，所以你不能再直接使用原始的键。

use std::collections::HashMap;

fn main() {
    let mut scores = HashMap::new();
    let team_name = String::from("Blue");

    scores.insert(team_name, 10);  // 键被转移到哈希映射中

    // 这里无法再使用 team_name，因为它的所有权已被转移
    // println!("{}", team_name); // 会编译失败
}

如果你想在插入后仍然保留键的使用权，可以使用引用作为键，但这要求值的生命周期必须比哈希映射长。

特别地

访问 HashMap 的键时需要使用引用 (&k)，而不是直接使用值 (k) 的原因与 Rust 的所有权和借用规则有关。

fn remove_element_in_hashmap(map:&mut HashMap<String,u32>,k:String){
    map.remove(&k);
    println!("{:?}",map);
}

在remove_element_in_hashmap(map:&mut HashMap<String,u32>,k:String)中，k 是传递给函数 get 的一个 String 类型的值。由于 String 是堆分配的数据类型，它在传递时会移动所有权（除非你传递引用）。

  • 如果你直接传递 k 而没有使用引用，它会移动所有权，意味着在 get 调用之后，k 变量将不再有效，因为它的所有权已经被转移到 HashMap::remove 函数。
  • 如果你传递 &k，这只是对 k 的一个借用，借用不会改变 k 的所有权，因此你可以继续使用 k。

总结

哈希映射是 Rust 中非常强大的数据结构，能够帮助我们存储键值对，并高效地进行查找、插入、更新、删除等操作。它提供了灵活的 API，可以方便地处理各种需求。
