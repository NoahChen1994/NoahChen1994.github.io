vec.len() 方法详解

vec.len() 方法用于获取 Vec<T> 中的元素个数，它不会改变 vec 的所有权，也不会影响 vec 的可变性。

fn main() {
    let vec = vec![1, 2, 3]; // 创建一个 Vec<i32>
    let vec_len = vec.len(); // 获取 vec 的长度
    println!("vec 长度: {}", vec_len);
    println!("vec 内容: {:?}", vec);
}

vec[index] 访问元素行为

  • vec[index] 访问的是 vec 中指定索引的元素，这是一个不可变借用。
  • 访问单个元素不会转移 vec 的所有权，也不会影响 vec 的可变性。

示例代码：

fn main() {
    let mut vec = vec![1, 2, 3];
    let x = vec[0]; // 借用 vec 的第一个元素（不可变借用）
    println!("{}", x);
    vec.push(4); // `vec` 仍然可用，可以继续修改
}

如果需要修改元素，需要使用可变引用：

fn main() {
    let mut vec = vec![1, 2, 3];
    vec[0] = 10; // 直接修改 vec[0] 的值
    println!("{:?}", vec);
}

或者使用 get_mut() 进行安全访问（避免索引越界）：

fn main() {
    let mut vec = vec![1, 2, 3];
    if let Some(first) = vec.get_mut(0) { // 获取可变引用
        *first = 10; // 通过解引用修改值
    }
    println!("{:?}", vec);
}

for e in &vec 遍历行为

在 for e in &vec 循环中，e 是 &T 类型，即对 vec 元素的不可变借用。

示例代码：

use std::collections::HashMap;

fn main() {
    let vec = vec![1, 2, 2, 3, 3, 3];
    let mut map = HashMap::new();
    let mut sum = 0;
    let mut mode = 0;

    for e in &vec { // `e` 是 `&i32` 类型，即不可变借用
        let count = map.entry(e).or_insert(0); // 获取或插入 0，返回 `&mut i32`
        *count += 1; // 需要解引用 `count` 才能增加值
        sum += e; // 自动解引用 `&i32`，直接进行加法

        //mode = e; // 报错
        mode = *e; // 需要手动解引用 `e` 才能赋值给 mode
    }
}

或者使用模式匹配 &e 直接解引用：

for &e in &vec { // `&e` 直接匹配 `i32`，无需手动解引用
    let count = map.entry(e).or_insert(0);
    *count += 1;
    sum += e;
    mode = e;
}

*count += 1; 详解

*count += 1; 语法用于修改 HashMap 中存储的值，count 是一个可变引用 &mut i32，所以需要解引用 *count 才能修改值。

示例代码：

use std::collections::HashMap;

fn main() {
    let mut map = HashMap::new();
    let key = "apple";
    let count = map.entry(key).or_insert(0); // `or_insert(0)` 返回 `&mut i32`
    *count += 1; // 需要解引用 `count` 才能修改值
    println!("{:?}", map); // 输出 {"apple": 1}
}

如果不加 *，会导致编译错误：

count += 1; // ❌ 编译错误：expected integer, found `&mut i32`
