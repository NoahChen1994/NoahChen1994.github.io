Rust中的泛型

Awsome
Rust

阅读时长:   6 分

Rust 允许我们使用泛型Generics为函数签名或结构体等项创建定义，这样它们就可以用于多种不同的具体数据类型。让我们看看如何使用泛型定义函数、结构体、枚举和方法，然后我们将讨论泛型如何影响代码性能。

在函数定义中使用泛型

下面展示了两个函数，它们的功能都是寻找 slice 中最大值。

src/main.rs

fn largest_i32(list: &[i32]) -> i32 {
    let mut largest = list[0];

    for &item in list.iter() {
        if item > largest {
            largest = item;
        }
    }

    largest
}

fn largest_char(list: &[char]) -> char {
    let mut largest = list[0];

    for &item in list.iter() {
        if item > largest {
            largest = item;
        }
    }

    largest
}

fn main() {
    let number_list = vec![34, 50, 25, 100, 65];

    let result = largest_i32(&number_list);
    println!("The largest number is {}", result);

    let char_list = vec!['y', 'm', 'a', 'q'];

    let result = largest_char(&char_list);
    println!("The largest char is {}", result);
}

在寻找 slice 中最大 char 时，我们的 largest_char 和 largest_num 逻辑完全一致。为避免代码重复，可以引入泛型参数，使函数更通用。

定义泛型函数时，原本指定类型的地方改用泛型表示，提高代码复用性和适应性。例如：

fn largest<T>(list: &[T]) -> T {

这里，T 代表泛型类型，符合 Rust 以单字母命名泛型参数的惯例（T 是 “type” 的缩写）。类型参数需在使用前声明，因此 <T> 位于函数名后，表示 largest 适用于任意 T 组成的 slice，并返回 T 类型的值。

下面是一个可以用于任何实现了 PartialOrd 和 Copy trait 的泛型的 largest 函数

// 在 T 的 trait bound 中指定 PartialOrd 以进行比较，同时 Copy 允许复制值
fn find_largest<T: PartialOrd + Copy>(list: &[T]) -> T {
    let mut largest = list[0];
    for &e in list.iter() {
        if e > largest {
            largest = e;
        }
    }
    largest
}

fn main() {
    let nums = vec![1, 2, 3];
    let chars = vec!['a', 'b', 'c', 'd'];
    println!("largest num is {}", find_largest(&nums)); // 输出: largest num is 3
    println!("largest char is {}", find_largest(&chars)); // 输出: largest char is d
}

泛型与结构体

同样也可以用 <> 语法来定义结构体，它包含一个或多个泛型参数类型字段。下面展示了如何定义和使用一个可以存放任何类型的 x 和 y 坐标值的结构体 Point：

// 定义一个 Point 结构体，x 和 y 具有相同类型 T
struct Point<T> {
    x: T,
    y: T,
}

// 定义一个 Point1 结构体，x 和 y 可以是不同类型的泛型
struct Point1<T, U> {
    x: T,
    y: U,
}

fn main() {
    let p = Point { x: 5, y: 10 };
    let p1 = Point1 { x: "hello", y: 3.14 };
    println!("Point: x = {}, y = {}", p.x, p.y); // 输出: Point: x = 5, y = 10
    println!("Point1: x = {}, y = {}", p1.x, p1.y); // 输出: Point1: x = hello, y = 3.14
}

泛型枚举

和结构体类似，枚举也可以在成员中存放泛型数据类型。例如我们常用的标准库提供的 Option<T> 枚举

// 泛型枚举定义
enum Option<T> {
    Some(T),
    None,
}

enum Result<T, E> {
    Ok(T),
    Err(E),
}

fn main() {
    let some_number = Option::Some(5);
    let some_text = Option::Some("hello");
    let result_ok: Result<i32, &str> = Result::Ok(200);
    let result_err: Result<i32, &str> = Result::Err("Error occurred");

    if let Option::Some(value) = some_number {
        println!("Some number: {}", value); // 输出: Some number: 5
    }
    if let Result::Ok(value) = result_ok {
        println!("Success: {}", value); // 输出: Success: 200
    }
}

方法定义中的泛型

注意必须在 impl 后面声明 T，这样就可以在 Point<T> 上实现的方法中使用它了

struct Point1<T, U> {
    x: T,
    y: U,
}

impl<T, U> Point1<T, U> {
    // 返回 x 的引用，T 类型
    fn x(&self) -> &T {
        &self.x
    }

    // 返回 y 的引用，U 类型
    fn y(&self) -> &U {
        &self.y
    }

    // fill 方法，结合两个不同类型的 Point1 结构体生成新结构体
    fn fill<V, W>(&self, other: &Point1<V, W>) -> Point1<T, W>
    where
        T: Clone,
        W: Clone,
    {
        Point1 {
            x: self.x.clone(),
            y: other.y.clone(),
        }
    }
}

fn main() {
    let p = Point1 { x: 1, y: 0.5 };
    let p2 = Point1 { x: 1.0, y: "hello".to_string() };
    println!("p:x is {}, y is {}", &p.x(), &p.y()); // 输出: p:x is 1, y is 0.5
    let filled = p.fill(&p2);
    println!("filled x is {}, y is {}", &filled.x(), &filled.y()); // 输出: filled x is 1, y is hello
}

泛型代码的性能

Rust 实现了泛型，使得使用泛型类型参数的代码相比使用具体类型并没有任何速度上的损失。

让我们看看一个使用标准库中 Option 枚举的例子：

let integer = Some(5);
let float = Some(5.0);

当 Rust 编译这些代码的时候，它会进行单态化。编译器会读取传递给 Option<T> 的值并发现有两种 Option<T>：一个对应 i32 另一个对应 f64。为此，它会将泛型定义 Option<T> 展开为 Option_i32 和 Option_f64，接着将泛型定义替换为这两个具体的定义。

编译器生成的单态化版本的代码看起来像这样，并包含将泛型 Option<T> 替换为编译器创建的具体定义后的用例代码：

src/main.rs

enum Option_i32 {
    Some(i32),
    None,
}

enum Option_f64 {
    Some(f64),
    None,
}

fn main() {
    let integer = Option_i32::Some(5);
    let float = Option_f64::Some(5.0);
}

我们可以使用泛型来编写不重复的代码，而 Rust 将会为每一个实例编译其特定类型的代码。这意味着在使用泛型时没有运行时开销；当代码运行，它的执行效率就跟好像手写每个具体定义的重复代码一样。这个单态化过程正是 Rust 泛型在运行时极其高效的原因。
