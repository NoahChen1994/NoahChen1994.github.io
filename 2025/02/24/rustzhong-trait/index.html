1. 什么是 Trait？

在 Rust 中，trait 类似于其他语言中的接口（interface）。它定义了一组方法签名，供实现该 trait 的类型定义具体行为。

1.1 定义 Trait

使用trait关键字,trait 定义是一种将方法签名组合起来的方法，目的是定义一个实现某些目的所必需的行为的集合。

// 定义一个 trait
pub trait Summary {
    fn summarize(&self) -> String;
}

Summary trait 规定了实现者必须提供 summarize 方法。

1.2 为类型实现 Trait

现在我们定义了 Summary trait，接着就可以在多媒体聚合库中需要拥有这个行为的类型上实现它了。下面展示了 NewsArticle 结构体上 Summary trait 的一个实现。

一旦实现了 trait，我们就可以用与 NewsArticle实例的非 trait 方法一样的方式.summarize()调用 trait 方法了：

use trait_demo::Summary;

pub struct NewArticle {
    pub title: String,
    pub headline: String,
    pub content: String,
    pub author: String,
}

impl Summary for NewArticle {
    fn summarize(&self) -> String {
        format!("{}, {}, ({})", self.title, self.headline, self.content)
    }
}

fn main() {
    let article = NewArticle {
        title: String::from("Rust Traits"),
        headline: String::from("Understanding Rust"),
        content: String::from("Rust provides powerful trait functionality."),
        author: String::from("Alice"),
    };

    println!("{}", article.summarize()); // Rust Traits, Understanding Rust, (Rust provides powerful trait functionality.)
}

2. Trait 的默认实现

有时为 trait 中的某些或全部方法提供默认的行为，而不是在每个类型的每个实现中都定义自己的行为是很有用的。这样当为某个特定类型实现 trait 时，可以选择保留或重载每个方法的默认行为。

pub trait Summary {
    fn summarize(&self) -> String {
        String::from("Summary default implementation!")
    }
}

pub struct NewArticle {
    pub title: String,
    pub headline: String,
    pub content: String,
    pub author: String,
}

impl Summary for NewArticle {}

fn main() {
    let article = NewArticle {
        title: String::from("Rust Traits"),
        headline: String::from("Understanding Rust"),
        content: String::from("Rust provides powerful trait functionality."),
        author: String::from("Alice"),
    };

    println!("{}", article.summarize()); // Summary default implementation!
}

3. Trait 作为参数

对于 item 参数，我们指定了 impl 关键字和 trait的 名称Summary，而不是具体的类型。该参数支持任何实现了指定 trait 的类型

pub trait Summary {
    fn summarize(&self) -> String;
}

pub struct NewArticle {
    pub title: String,
    pub headline: String,
    pub content: String,
    pub author: String,
}

 impl Summary for NewsArticle {
     fn summarize(&self) -> String {
          format!("{},{},({})",self.title, self.headline, self.content)
     }
}

//Trait 作为参数
pub fn notify(item: &impl Summary) {
    println!("Breaking news! {}", item.summarize());
}

fn main() {
    struct NewsArticle;

    let article = NewArticle {
        title: String::from("Rust Traits"),
        headline: String::from("Understanding Rust"),
        content: String::from("Rust provides powerful trait functionality."),
        author: String::from("Alice"),
    };
    notify(&article); // Breaking news! How to Ask a quesstion?,Ask a quesstion,(content)
}

4. Trait Bound

impl Trait 语法适用于直观的例子，它实际上是一种较长形式语法的语法糖。我们称为 trait bound，

4.1. Trait Bound 语法

T: Trait 是 trait bound 的基本语法，表示泛型参数 T 必须实现 Trait。例如：

pub fn notify<T: Summary>(item: T) {
    println!("Breaking news! {}", item.summarize());
}

4.2. impl Trait 语法

impl Trait 是 trait bound 的一种简洁形式，很方便，适用于短小的例子。trait bound 则适用于更复杂的场景。例如，可以获取两个实现了 Summary 的参数。使用 impl Trait 的语法看起来像这样：

pub fn notify(item1: impl Summary, item2: impl Summary)

这适用于 item1 和 item2 允许是不同类型的情况（只要它们都实现了 Summary）。不过如果你希望强制它们都是相同类型呢？这只有在使用 trait bound 时才有可能：

pub fn notify<T: Summary>(item1: T, item2: T)

4.3. 多个 Trait Bound

如果 notify 需要显示 item 的格式化形式，同时也要使用 summarize 方法，那么 item 就需要同时实现两个不同的 trait：Display 和 Summary。这可以通过 + 语法实现：

// 可以通过 `+` 来指定多个 trait：
pub fn notify(item: impl Summary + Display) {

+ 语法也适用于泛型的 trait bound：

pub fn notify<T: Summary + Display>(item: T) {

通过指定这两个 trait bound，notify 的函数体可以调用 summarize 并使用 {} 来格式化 item。

4.4. 通过 where 简化 trait bound

使用过多的 trait bound 也有缺点。每个泛型有其自己的 trait bound，所以有多个泛型参数的函数在名称和参数列表之间会有很长的 trait bound 信息，这使得函数签名难以阅读

fn some_function<T: Display + Clone, U: Clone + Debug>(t: T, u: U) -> i32 {

当函数的泛型参数有多个 trait bound 时，可以使用 where 从句来简化签名：

fn some_function<T, U>(t: T, u: U) -> i32
    where T: Display + Clone,
          U: Clone + Debug
{

这个函数签名就显得不那么杂乱，函数名、参数列表和返回值类型都离得很近，看起来跟没有那么多 trait bounds 的函数很像

4.5. Trait 作为函数返回值

使用 impl Trait 可以返回实现某个 trait 的类型，而不暴露具体类型：

fn trait_as_return() -> impl Summary{
    NewArticle{
        title:String::from("Deeping Thinking in Rust"),
        headline:String::from("Rust"),
        content:String::from("content"),
        author:String::from("Micel")
    }
}

通过使用 impl Summary 作为返回值类型，我们指定了 returns_summarizable 函数返回某个实现了 Summary trait 的类型，但是不确定其具体的类型。在这个例子中 returns_summarizable 返回了一个 Tweet，不过调用方并不知情。

4.6. 有条件地实现方法
使用 trait bound 在泛型实现中条件性地定义方法。例如，Pair<T> 类型只有在 T 实现了 Display 和 PartialOrd 时，才会实现 cmp_display 方法：

use std::fmt::Display;

struct Pair<T>{
    x:T,
    y:T
}

impl <T> Pair<T>{
    fn new(x:T, y:T)->Self{
        Self{x,y}
    }
}

//只有当 T 实现了 `Display + PartialOrd`，才会实现 `cmp_display` 方法
impl <T: Display + PartialOrd> Pair<T>{
    fn cmp_display(&self){
        if self.x>self.y{
            println!("The largest element is: {}",self.x);
        }{
            println!("The largest element is: {}",self.y);
        }
    }
} 

fn main(){
    let pair1 = Pair{x:1,y:2};
    pair1.cmp_display();//The largest element is: 2

    let pair2 = Pair::new("apple","banana");
    pair2.cmp_display();//The largest element is: banana

    let pair3 = Pair::new(vec![1, 2, 3], vec![4, 5, 6]);
    pair3.cmp_display(); // ^^^^^^^^^^^ method cannot be called on `Pair<Vec<{integer}>>` due to unsatisfied trait bounds
}

通过 trait 和 trait bound，Rust 能够在编译时确保类型行为的正确性，避免了运行时错误，从而提升了性能和安全性。这也为泛型编程提供了强大的支持，使得代码更加灵活且易于维护。
