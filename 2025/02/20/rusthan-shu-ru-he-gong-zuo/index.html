Rust 函数如何工作

1. 引言

在 Rust 中，函数是程序的基本构建块之一，允许我们封装一段代码，使其能够多次调用。Rust 提供了多种不同形式的函数，包括带参数和返回值的函数。理解函数的工作方式是编写 Rust 程序的基础。

2. 定义函数

Rust 中的函数使用 fn 关键字定义。函数可以有参数，返回值，也可以没有这些元素。

2.1 基本函数定义

最简单的函数定义如下：

fn main() {
    println!("Hello, world!");  // main 函数是程序的入口
}

2.2 带参数的函数

函数可以接受参数，参数列表放在圆括号中，并通过逗号分隔：

fn greet(name: &str) {
    println!("Hello, {}!", name);
}

fn main() {
    greet("Rust");  // 调用 greet 函数并传递参数
}

在这个例子中，greet 函数接受一个类型为 &str（字符串切片）的参数。

2.3 带返回值的函数

函数也可以返回值。返回值的类型在函数签名中指定，使用 -> 符号表示返回类型：

fn add(x: i32, y: i32) -> i32 {
    x + y
}

fn main() {
    let sum = add(5, 3);  // 调用 add 函数并获取返回值
    println!("The sum is: {}", sum);
}

上面的例子中，add 函数接受两个 i32 类型的参数，并返回它们的和，返回值类型为 i32。

3. 函数返回值

Rust 中的函数可以返回一个值，但返回值的类型必须与函数签名中的返回类型一致。可以使用 return 关键字显式地返回值，也可以通过表达式隐式返回。

3.1 使用 return 返回值

fn square(x: i32) -> i32 {
    return x * x;  // 显式返回值
}

3.2 隐式返回值

fn square(x: i32) -> i32 {
    x * x  // 最后一行表达式的值会自动作为返回值
}

4. 函数参数

Rust 中的函数参数可以是多种类型，包括值类型和引用类型。函数调用时，参数会根据传入的值进行传递，Rust 支持两种类型的参数传递：按值传递和按引用传递。

4.1 按值传递

按值传递意味着将参数的值复制到函数内部。

fn double(x: i32) -> i32 {
    x * 2
}

fn main() {
    let num = 5;
    let result = double(num);  // 将 num 的值传递给 double 函数
    println!("Doubled value: {}", result);
}

4.2 按引用传递

按引用传递允许函数通过引用访问参数，而不复制它。这样可以避免不必要的内存开销。

fn print_length(s: &str) {
    println!("Length: {}", s.len());
}

fn main() {
    let text = String::from("Hello, Rust!");
    print_length(&text);  // 传递引用而不是值
}

通过使用 & 符号，我们将 text 的引用传递给函数，函数不会获取 text 的所有权，只是借用它。

5. 返回多个值

Rust 支持返回多个值。最常见的做法是使用元组将多个值打包一起返回。

fn calculate(x: i32, y: i32) -> (i32, i32) {
    (x + y, x * y)
}

fn main() {
    let result = calculate(5, 3);
    println!("Sum: {}, Product: {}", result.0, result.1);
}

6.如何理解函数入参值传递和引用传递？

在 Rust 中，fn print_length(s: &str) 和 fn print_length(s: String) 都是定义函数时对参数类型的不同选择，它们的区别在于如何处理传入的字符串（即它们如何处理内存和所有权）。

让我们分别详细理解这两种写法：

6.1. 使用引用类型

fn print_length(s: &str) {
    println!("Length: {}", s.len());
}
  • &str 是一个字符串切片的引用，代表对一个已经存在的字符串的借用，而不是获取字符串的所有权。
  • 引用意味着我们不需要将 s 的数据从原来的位置复制过来。它只是对外部数据的一个“借用”。
  • 这种方式 不会获取所有权，所以在函数执行结束后，原始的数据仍然有效，调用者不需要担心数据的生命周期问题。
  • 传递 &str 是按引用传递，不会发生内存复制，效率较高。

适用场景：

  • 适用于你不需要修改传入的字符串，并且不需要拥有它的所有权时。
  • 如果你只需要读取字符串的内容（比如计算长度），而不关心它的生命周期管理，使用 &str 是更好的选择。

6.22. 使用拥有所有权的 String

fn print_length(s: String) {
    println!("Length: {}", s.len());
}
  • String 是一个动态大小的、堆分配的字符串类型，拥有自己的所有权。
  • 当你传递 String 时，函数会 获取 s 的所有权，这意味着传入的 String 被移动到函数内部，调用者将无法再访问该字符串。
  • 由于拥有所有权，函数内部可以修改 String 的内容，甚至在函数中销毁它。
  • 如果函数结束，所有权将被销毁，内存会被释放。

适用场景：

  • 适用于你需要传递并修改或销毁字符串的情况。如果你打算改变或消费传入的字符串（例如修改或将其移交给其他函数），使用 String 是合适的。

对比总结

特性 &str String
传递方式 按引用传递 按值传递（所有权转移）
内存 不会复制数据，只是借用 在堆上分配内存，具有所有权
修改 不可以修改（不可变） 可以修改（可变）
所有权 不拥有所有权 拥有所有权
效率 更高（避免了数据复制） 较低（可能会有内存复制）

举个例子

fn main() {
    let s1 = String::from("Hello, world!");
    let s2 = &s1;  // s2 是对 s1 的引用
    print_length(s2);  // 传递引用
    println!("{}", s1); // 仍然可以使用 s1，因为没有转移所有权
}

fn print_length(s: &str) {
    println!("Length: {}", s.len());
}

在这个例子中，s2 是 s1 的引用，因此调用 print_length(s2) 后，s1 仍然有效，因为所有权没有被转移。

但是，如果我们使用 String 类型：

fn main() {
    let s1 = String::from("Hello, world!");
    print_length(s1);  // 传递所有权
    // println!("{}", s1); // 这将报错，因为 s1 的所有权已经转移给了 print_length
}

fn print_length(s: String) {
    println!("Length: {}", s.len());
}

在这个例子中，s1 的所有权被转移到 print_length 函数中，因此调用 print_length 后，s1 不再有效。如果尝试在 main 中使用 s1，编译器会报错。

总结

  • &str 是对字符串的借用，适用于不需要修改或获取所有权的情况。
  • String 是拥有所有权的字符串，适用于需要转移所有权或修改字符串内容的情况。

7. 小结

  • 函数通过 fn 关键字定义，可以接受参数并返回值。
  • 函数的参数可以是值类型，也可以是引用类型，按值传递和按引用传递都支持。
  • 函数返回值时，Rust 提供了显式和隐式返回两种方式。
  • Rust 支持返回多个值，通常通过元组来实现。
  • 理解函数入参值传递和引用传递。
