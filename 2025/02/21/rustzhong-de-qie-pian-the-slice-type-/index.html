Rust中切片 （The slice type）

Awsome
Rust
发布日期:   2025-02-21
更新日期:   2025-02-21
文章字数:   1.4k
阅读时长:   5 分

切片是 Rust 中的一个重要概念，它提供了一种引用数据部分的方式，而无需复制数据本身。切片允许你对集合（如数组、Vec、String）的一部分进行操作，同时保持对原始数据的引用。

1. 切片的基本概念

在 Rust 中，切片（slice）是一种对集合的一部分的引用，它不拥有数据的所有权，而是充当数据的一扇“窗口”。切片的主要类型有：

  • 数组切片（Array Slice）：引用数组的一部分。
  • 字符串切片（String Slice）：引用字符串的一部分。

切片默认是不可变引用，但可以显式声明为可变切片，允许修改数据。

2. 切片的语法

切片使用 & 表示引用，并通过 区间 [start..end] 设定切片范围。

2.1 数组切片

let arr = [1, 2, 3, 4, 5];
let slice = &arr[1..3];  // 这个切片包含 arr[1] 和 arr[2]，即 [2, 3]
println!("{:?}", slice); // 输出: [2, 3]

[1..3] 代表索引 1 到 3（不含 3），即 arr[1] 和 arr[2] 组成的切片。

2.2 字符串切片

字符串切片是 &str 类型，它允许你高效地引用 String 的一部分：

let s = String::from("hello");
let slice = &s[0..2];  // 切片包含 "he"
println!("{}", slice); // 输出: he

⚠ 注意：

  • Rust 的 String 使用 UTF-8 编码，必须保证切片范围是 有效的字符边界，否则会导致运行时错误。

3. 切片的特点

3.1 默认不可变

切片默认是不可变的，意味着你不能通过不可变切片修改原始数据：

let v = vec![1, 2, 3];
let slice = &v[0..2]; // 不可变切片
// slice[0] = 10; // ❌ 编译错误：无法修改不可变切片

3.2 可变切片

如果需要修改数据，可以使用 可变切片 &mut [T]：

let mut v = vec![1, 2, 3];
let slice = &mut v[0..2]; // 可变切片
slice[0] = 10;  // ✅ 允许修改
println!("{:?}", v); // 输出: [10, 2, 3]

3.3 切片的存储方式

  • 切片 本质上是一个指针和一个长度，指向原始数据的一部分，而不是拷贝数据本身。
  • 通过切片访问的数据不会被复制，因此切片比完整拷贝更高效。

4. 切片 vs. 数组

特性 数组 [T; N] 切片 &[T]
所有权 拥有数据 仅引用数据
大小 固定大小 动态大小
存储 栈上存储 指向原始数据
修改 可以修改 只有可变切片可修改

数组是固定大小的，而切片是动态大小的引用，它可以灵活地引用数组的一部分，而不影响数组本身。

5. 切片 vs. Vec

切片也可以用于 Vec（动态数组）：

let v = vec![1, 2, 3, 4, 5];
let slice = &v[1..4];  // 索引 1 到 4（不含 4）
println!("{:?}", slice); // 输出: [2, 3, 4]

区别：

  • Vec 拥有数据，可以动态增长或缩小。
  • 切片只是 Vec 的引用，无法修改 Vec 的长度。

6. 为什么使用切片？

使用切片有多个优点：

✅ 避免数据复制：切片是引用，减少不必要的数据拷贝。
✅ 高效内存管理：切片不会创建新数据，而是直接引用原数据，提高性能。
✅ 灵活性：可以在不同的函数之间传递数据的一部分，而不会影响原始数据的所有权。

7. 切片的使用示例

fn first_word(s: &String) -> &str {
    let bytes = s.as_bytes();

    for (i, &item) in bytes.iter().enumerate() {
        if item == b' ' { // 遇到空格
            return &s[0..i]; // 返回切片
        }
    }

    &s[..] // 返回整个字符串
}

fn main() {
    let s = String::from("hello world");
    let word = first_word(&s);
    println!("第一个单词: {}", word); // 输出: hello
}

解释

  • 这个函数 first_word 返回字符串中的第一个单词，而不会获取所有权。
  • &s[0..i] 返回 字符串切片，避免拷贝整个 String，提高效率。
  • &s[..] 是完整字符串的切片。

8. 需要注意的事项

8.1 切片范围必须有效

let s = String::from("你好");
let slice = &s[0..1]; // ❌ 运行时错误：无效 UTF-8 切片

🚨 Rust 使用 UTF-8 编码，所以切片必须落在字符边界上，否则会触发错误。

8.2 生命周期问题

切片的生命周期通常与原始数据一致：

fn invalid_slice() -> &str {
    let s = String::from("hello");
    &s[..] // ❌ 编译错误：s 在函数结束时被释放，切片变成悬垂引用
}

✅ 正确的做法：

fn valid_slice(s: &String) -> &str {
    &s[..] // ✅ 传入 `s` 的引用，避免悬垂引用
}

9. 总结

✅ 切片是一种高效的方式来引用数组或字符串的一部分，而不会影响原始数据的所有权。
✅ 默认是不可变引用，若需要修改数据，需要使用可变切片 &mut [T]。
✅ 切片比完整拷贝更节省内存，并且在函数间传递时更高效。
✅ 字符串切片必须落在 UTF-8 边界上，否则会导致运行时错误。
✅ 切片的生命周期与原始数据一致，不能返回指向局部变量的切片，否则会导致悬垂引用。

切片是 Rust 内存管理的重要工具，熟练掌握它可以让你的代码更加安全、高效！🚀
