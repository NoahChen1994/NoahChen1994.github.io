一个使用结构体的示例程序

为了理解何时会需要使用结构体，让我们编写一个计算长方形面积的程序。我们会从单独的变量开始，接着重构程序直到使用结构体替代他们为止。

1. 结构体的基本使用

示例程序的目标是计算一个矩形的面积。最初，直接使用 单独的变量 来表示矩形的长宽：

fn main() {
    let width = 30;
    let height = 50;

    println!(
        "The area of the rectangle is {} square pixels.",
        area(width, height)
    );
}

fn area(width: u32, height: u32) -> u32 {
    width * height
}
  • 这种方式的问题是：
    • 长和宽是 独立变量，它们之间的关系（表示同一个矩形）并不明显。
    • 代码的 可读性和可维护性较差。

2. 使用元组（Tuple）优化

可以用元组存储长宽信息：

fn main() {
    let rect = (30, 50);

    println!(
        "The area of the rectangle is {} square pixels.",
        area(rect)
    );
}

fn area(dimensions: (u32, u32)) -> u32 {
    dimensions.0 * dimensions.1
}

优缺点

✅ 优点：

  • 通过元组 (u32, u32) 将长宽绑定在一起，使得数据更有逻辑性。

❌ 缺点：

  • dimensions.0 和 dimensions.1 缺乏语义信息，不清楚 .0 是长还是宽，代码的 可读性不佳。

3. 使用结构体（Struct）优化

使用 struct 定义矩形，提高代码的可读性：

struct Rectangle {
    width: u32,
    height: u32,
}

fn main() {
    let rect = Rectangle {
        width: 30,
        height: 50,
    };

    println!(
        "The area of the rectangle is {} square pixels.",
        area(&rect)
    );
}

fn area(rect: &Rectangle) -> u32 {
    rect.width * rect.height
}

为什么传 &Rectangle 而不是 Rectangle？

  • 传递整个 Rectangle 会发生所有权转移，导致 main 函数无法继续使用 rect。
  • 传递 &Rectangle 只传递引用，不会发生所有权转移，提高代码复用性。

4. Debug Trait 让结构体可打印

默认情况下，结构体 不能被 println! 直接打印。如果尝试：

println!("rect is {}", rect); // ❌ 错误 error[E0277]: `Rectangle` doesn't implement `std::fmt::Display`

解决方案：使用 #[derive(Debug)]

可以让结构体派生 Debug trait：

#[derive(Debug)]
struct Rectangle {
    width: u32,
    height: u32,
}

fn main() {
    let rect = Rectangle {
        width: 30,
        height: 50,
    };

    println!("rect is {:?}", rect); // 单行调试输出
    println!("rect is {:#?}", rect); // 更美观的格式化输出
}

输出：

rect is Rectangle { width: 30, height: 50 }
rect is Rectangle {
    width: 30,
    height: 50,
}

✅ Debug 让结构体支持 println! 调试输出，{:#?} 还能 格式化输出。

5. 使用 dbg! 进行调试

Rust 提供了 dbg! 宏，它不仅可以输出变量，还会 返回变量值，适用于调试：

fn main() {
    let rect = Rectangle {
        width: 30,
        height: 50,
    };

    dbg!(&rect);
}

输出：

[src/main.rs:10] &rect = Rectangle {
    width: 30,
    height: 50,
}

✅ dbg!(&rect) 不仅打印结构体值，还附带文件名和代码行号，适合 临时调试。

6. 结构体方法（impl 关键字）

前面 area(&Rectangle) 计算面积，但 函数和结构体是分开的，可以使用 impl 关键字把方法绑定到结构体：

#[derive(Debug)]
struct Rectangle {
    width: u32,
    height: u32,
}

impl Rectangle {
    fn area(&self) -> u32 {
        self.width * self.height
    }
}

fn main() {
    let rect = Rectangle {
        width: 30,
        height: 50,
    };

    println!("The area of the rectangle is {} square pixels.", rect.area());
}

✅ 方法 area 被绑定到 Rectangle 结构体，调用时使用 rect.area()，提高了可读性。

7. 方法可以有多个参数

可以在 impl 里定义多个方法：

impl Rectangle {
    fn area(&self) -> u32 {
        self.width * self.height
    }

    fn can_hold(&self, other: &Rectangle) -> bool {
        self.width > other.width && self.height > other.height
    }
}

fn main() {
    let rect1 = Rectangle { width: 30, height: 50 };
    let rect2 = Rectangle { width: 10, height: 40 };
    let rect3 = Rectangle { width: 60, height: 45 };

    println!("Can rect1 hold rect2? {}", rect1.can_hold(&rect2));
    println!("Can rect1 hold rect3? {}", rect1.can_hold(&rect3));
}

✅ can_hold 方法接收另一个 Rectangle，判断当前矩形是否能容纳它。

8. 关联函数（new 作为构造函数）

可以在 impl 中 定义不依赖 self 的方法，类似于 静态方法，常用于创建实例：

impl Rectangle {
    fn new(width: u32, height: u32) -> Rectangle {
        Rectangle { width, height }
    }
}

fn main() {
    let rect = Rectangle::new(30, 50);
    println!("Rectangle: {:#?}", rect);
}

✅ Rectangle::new(30, 50) 类似构造函数，让代码更加直观。

9. 结构体方法 vs 普通函数

特性 普通函数 impl 方法
作用域 结构体外部 结构体内部
调用方式 area(&rect) rect.area()
访问字段 rect.width self.width
适用场景 结构简单、临时操作 结构较复杂、操作与数据绑定

✅ 如果方法逻辑和结构体强相关，建议使用 impl 绑定，提高可读性和封装性。

总结

  • 使用 结构体封装数据，提高代码可读性。
  • #[derive(Debug)] 让结构体支持 println! 输出，dbg! 适用于调试。
  • impl 绑定方法到结构体，提高封装性。
  • 方法接收 &self，避免所有权转移，提高复用性。
  • 关联函数 new() 作为构造方法，简化实例化。

Rust 结构体和方法的结合，使代码更加 清晰、模块化、可读性强。🚀
