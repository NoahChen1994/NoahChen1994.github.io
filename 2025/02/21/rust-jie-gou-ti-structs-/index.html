Rust 结构体(Structs)

Awsome
Rust
发布日期:   2025-02-21
更新日期:   2025-02-21
文章字数:   1.3k
阅读时长:   5 分

结构体（struct）是 Rust 中用于组织相关数据的一种方式，类似于其他语言中的类（class）或结构体（struct）。Rust 的结构体可以帮助我们将多个值组合在一起，形成更具语义的复杂数据类型，从而提升代码的可读性和可维护性。

1. 结构体的基本定义

在 Rust 中，结构体使用 struct 关键字定义，通常包含多个字段，每个字段都有名称和类型。其基本语法如下：

struct User {
    username: String,
    email: String,
    sign_in_count: u64,
    active: bool,
}

这个结构体 User 由四个字段组成：

  • username: 存储用户名，类型为 String
  • email: 存储电子邮件，类型为 String
  • sign_in_count: 记录登录次数，类型为 u64
  • active: 记录用户是否活跃，类型为 bool

2. 结构体的实例化

定义结构体后，可以创建其实例：

fn main() {
    let user1 = User {
        username: String::from("alice"),
        email: String::from("alice@example.com"),
        sign_in_count: 1,
        active: true,
    };

    println!("Username: {}", user1.username);
}
  • 这里 user1 是 User 类型的实例，其中 username 和 email 需要用 String::from() 进行字符串的创建，因为 String 是 heap 分配的数据类型。

3. 访问和修改结构体字段

访问字段

结构体字段可以通过 . 访问：

println!("User email: {}", user1.email);

修改字段

如果 user1 不是 mut，它的字段无法修改。因此，修改结构体实例的字段需要先声明 mut：

fn main() {
    let mut user1 = User {
        username: String::from("alice"),
        email: String::from("alice@example.com"),
        sign_in_count: 1,
        active: true,
    };

    user1.email = String::from("new_email@example.com");

    println!("Updated email: {}", user1.email);
}
  • 由于 String 具有所有权（ownership），如果 user1.email 被修改，原来的值会被释放。
  • 但 结构体本身的字段不是独立的变量，因此不能只让单个字段是 mut，整个结构体都必须是 mut。

4. 结构体简化初始化

使用字段初始化简写（Field Init Shorthand）

如果变量名和字段名相同，可以省略重复部分：

fn build_user(username: String, email: String) -> User {
    User {
        username,
        email,
        sign_in_count: 1,
        active: true,
    }
}
  • username: username 和 email: email 被简写为 username 和 email。

结构体更新语法（Struct Update Syntax）

如果希望基于已有的结构体创建新的结构体实例，并更改其中的部分字段，可以使用 更新语法：

fn main() {
    let user1 = User {
        username: String::from("alice"),
        email: String::from("alice@example.com"),
        sign_in_count: 1,
        active: true,
    };

    let user2 = User {
        email: String::from("bob@example.com"),
        ..user1 // 其余字段与 `user1` 相同
    };

    println!("User2 email: {}", user2.email);
}
  • ..user1 表示 user2 继承 user1 的 sign_in_count 和 active 字段。
  • 注意：user1 被部分转移（move），因为 String 类型的 username 被 user2 复用，user1 无法再使用 username。
  • 如果 User 的字段是 Copy 类型（如 u32、bool），则 user1 仍然可以使用。

5. 元组结构体（Tuple Structs）

Rust 允许定义没有字段名称的结构体，称为 元组结构体：

struct Color(i32, i32, i32);
struct Point(i32, i32, i32);

fn main() {
    let black = Color(0, 0, 0);
    let origin = Point(0, 0, 0);

    println!("Black color: {}, {}, {}", black.0, black.1, black.2);
}
  • Color 和 Point 拥有相同的字段类型，但表示不同的概念。
  • 访问字段时使用索引，如 black.0。

元组结构体的适用场景：

  • 适用于字段数量较少，并且字段本身的含义较明显的情况。

  • 适用于封装特定数据但不需要明确字段名的场景，例如：

    struct ID(u32);
struct Coordinates(f64, f64);

6. 类单元结构体（Unit-like Structs）

如果一个结构体 没有任何字段，它类似于 空元组 ()：

struct AlwaysEqual;

fn main() {
    let _subject = AlwaysEqual;
}
  • 这种 “类单元结构体” 常用于 实现 trait，但不存储任何数据。

7. 结构体的所有权

结构体的字段可以是任何类型，包括 String 这样的 堆上分配 数据类型。因此，在 struct 设计时，需要考虑所有权问题：

struct User {
    username: String, // String 是堆上的数据
    email: String,
    sign_in_count: u64,
    active: bool,
}

fn main() {
    let user1 = User {
        username: String::from("alice"),
        email: String::from("alice@example.com"),
        sign_in_count: 1,
        active: true,
    };

    let user2 = user1; // ❌ 此时 `user1` 被 `move`，不能再使用

    println!("{}", user1.username); // ❌ 编译错误
}

  • 如何避免 move？

    • 使用 clone()：user2 = user1.clone();

    • 使用 &传递引用，避免所有权转移：

      fn display_email(user: &User) {
    println!("{}", user.email);
}

    • 让结构体使用 Copy 类型（如 u32），避免所有权问题。

8. 结构体 vs 元组 vs 枚举

特性 结构体 元组结构体 元组
字段有名字 ✅ ❌ ❌
适用于复杂数据 ✅ ⭕ ❌
可读性 ✅ ⭕ ❌
可变性 ✅ ✅ ✅
类型安全 ✅ ✅ ❌

总结

  • struct 用于封装相关数据，提高代码的可读性和结构化。
  • 结构体可以 使用 .. 语法更新字段，但要注意所有权问题。
  • 元组结构体 适用于没有必要命名字段的场景。
  • 类单元结构体 主要用于 实现 trait，而不存储数据。
  • 设计结构体时需要 考虑所有权和生命周期，避免 move 导致的问题。

Rust 结构体提供了一种强大而灵活的数据组织方式，使得代码更加清晰、可维护。后续可以结合 方法（methods） 和 trait 进一步扩展结构体的功能。🚀
