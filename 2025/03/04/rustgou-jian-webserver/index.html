本章节展示了如何使用Rust标准库中的功能实现一个简单的Web服务器，包括监听客户端请求、读取请求数据并返回HTTP响应。我们将分析两种实现：单线程服务器和多线程服务器。代码分为两部分：单线程版本直接处理请求，多线程版本使用线程池提高并发性能。

1.单线程Web服务器

单线程Web服务器通过绑定本地IP地址和端口，接收客户端的HTTP请求，并根据请求路径返回相应的HTML文件内容。它简单直接，但无法处理并发请求。

webserver/src/main.rs

// 引入标准库模块
use std::{
    fs, // 文件系统操作，用于读取文件内容
    io::{prelude::*, BufReader}, // 输入输出操作，包含读取流的缓冲读取器
    net::{TcpListener, TcpStream}, // 网络操作，处理TCP连接
};

// 主函数，程序入口
fn main() {
    // 绑定到本地地址 127.0.0.1:7788，创建 TCP 监听器
    // unwrap 处理绑定失败的错误（如端口被占用）
    let listener = TcpListener::bind("127.0.0.1:7788").unwrap();

    // 无限循环，处理传入的连接
    // incoming 返回一个迭代器，产生 Result<TcpStream, io::Error>
    for stream in listener.incoming() {
        // 获取每个连接的 TcpStream，unwrap 处理连接错误
        let stream = stream.unwrap();
        // 调用处理函数，处理客户端请求
        handle_msg(stream);
    }
}

// 处理客户端请求并发送响应的函数
// 参数 mut stream: TcpStream 表示可变的 TCP 流，用于读写
fn handle_msg(mut stream: TcpStream) {
    // 创建缓冲读取器，从 stream 中读取数据
    // &stream 是不可变引用，BufReader 用于高效逐行读取
    let buf_reader = BufReader::new(&stream);

    // 读取请求的第一行（请求行，例如 "GET / HTTP/1.1"）
    // lines() 返回行迭代器，next() 取第一行，unwrap 处理 Option 和 Result
    let request_line = buf_reader.lines().next().unwrap().unwrap();

    // 根据请求路径决定响应状态和文件名
    // 使用 if-else 判断，匹配根路径和其他路径
    let (status, file_name) = if request_line == "GET / HTTP/1.1" {
        // 请求根路径，返回 200 OK 和 hello.html
        ("HTTP/1.1 200 OK", "hello.html")
    } else {
        // 其他路径，返回 404 Not Found 和 404.html
        ("HTTP/1.1 404 NOT FOUND", "404.html")
    };

    // 读取指定文件的内容，unwrap 处理读取错误
    let content = fs::read_to_string(file_name).unwrap();

    // 计算内容字节长度，用于 Content-Length 头
    let length = content.len();

    // 构造 HTTP 响应，包括状态行、头和正文
    // \r\n 是 HTTP 协议的换行符
    let response = format!("{status}\r\nContent-Length: {length}\r\n\r\n{content}");

    // 将响应写入 TCP 流，as_bytes 转换为字节数组
    // write_all 发送所有字节，unwrap 处理写入错误
    stream.write_all(response.as_bytes()).unwrap();
}

运行流程

  1. 绑定地址：
    • TcpListener::bind("127.0.0.1:7788")创建监听器，绑定本地地址和端口。
  2. 监听连接：
    • listener.incoming()迭代器持续接收客户端连接，每次返回一个 Result<TcpStream, io::Error>。
  3. 处理请求：
    • handle_msg 接收 TcpStream，读取请求行。
    • 根据路径（GET / HTTP/1.1 或其他）选择状态和文件。
  4. 发送响应：
    • 读取文件，构造HTTP响应，写入流并发送。

重要代码讲解

  • let buf_reader = BufReader::new(&stream);：
    • 作用：创建缓冲读取器，提高读取效率。
    • 为什么用引用：&stream 借用 stream，避免转移所有权，确保后续写入可用。
  • request_line = buf_reader.lines().next().unwrap().unwrap();：
    • 分解：
      • lines()：返回行迭代器。
      • next()：返回 Option，取第一行。
      • 第一个 unwrap()：解包 Option，处理无行的情况。
      • 第二个 unwrap()：解包 Result，处理读取错误。
    • 风险：未处理错误，生产环境中应使用 match。
  • stream.write_all(response.as_bytes())：
    • 作用：将响应字节写入流。
    • 性能：as_bytes() 零成本转换，避免复制。

2.多线程Web服务器

多线程版本通过线程池（ThreadPool）实现并发处理，支持根路径（/）和延迟路径（/sleep），展示线程并发的优势。它将请求分发给多个线程，提高吞吐量。

webserver/src/lib.rs

// 引入标准库模块
use std::{
    sync::{mpsc, Arc, Mutex}, // 同步原语：通道、原子引用计数、互斥锁
    thread,                  // 线程操作
};

// 定义线程池结构体，管理一组工作者线程
pub struct ThreadPool {
    workers: Vec<Worker>,             // 存储工作者线程的向量
    sender: Option<mpsc::Sender<Job>>, // 发送任务的通道，可能为空
}

// 定义 Job 类型，表示线程池的任务
// Box<dyn FnOnce() + Send + 'static>：可运行一次的任务，线程安全，静态生命周期
type Job = Box<dyn FnOnce() + Send + 'static>;

// 为 ThreadPool 实现方法
impl ThreadPool {
    /// 创建新的线程池
    /// 参数 size: usize 表示线程池中的线程数量
    /// # Panics
    /// 如果 size 为 0，则 panic
    pub fn new(size: usize) -> ThreadPool {
        assert!(size > 0); // 确保线程数大于 0

        // 创建通道，返回发送端和接收端
        let (sender, receiver) = mpsc::channel();

        // 用 Arc 和 Mutex 包装接收端，允许多线程共享
        let receiver = Arc::new(Mutex::new(receiver));

        // 创建工作者向量，预分配容量
        let mut workers = Vec::with_capacity(size);

        // 初始化每个工作者线程
        for id in 0..size {
            // 克隆 receiver 的 Arc，传递给每个 Worker
            workers.push(Worker::new(id, Arc::clone(&receiver)));
        }

        ThreadPool {
            workers,           // 保存工作者
            sender: Some(sender), // 保存发送端
        }
    }

    /// 执行任务
    /// 参数 f: F 是闭包，满足 FnOnce、Send 和 'static 约束
    pub fn execute<F>(&self, f: F)
    where
        F: FnOnce() + Send + 'static,
    {
        // 将闭包装箱为 Job 类型
        let job = Box::new(f);

        // 通过通道发送任务，unwrap 处理发送错误
        self.sender.as_ref().unwrap().send(job).unwrap();
    }
}

// 为 ThreadPool 实现 Drop trait，清理资源
impl Drop for ThreadPool {
    fn drop(&mut self) {
        // 释放发送端，关闭通道
        drop(self.sender.take());

        // 遍历并关闭每个工作者线程
        for worker in &mut self.workers {
            println!("Shutting down worker {}", worker.id);

            // 取出线程句柄并等待线程结束
            if let Some(thread) = worker.thread.take() {
                thread.join().unwrap();
            }
        }
    }
}

// 定义 Worker 结构体，表示线程池中的工作者
struct Worker {
    id: usize,                   // 工作者的唯一标识
    thread: Option<thread::JoinHandle<()>>, // 线程句柄，可能为空
}

// 为 Worker 实现方法
impl Worker {
    // 创建新工作者
    // 参数 id: usize 是工作者ID，receiver 是共享的任务接收端
    fn new(id: usize, receiver: Arc<Mutex<mpsc::Receiver<Job>>>) -> Worker {
        // 启动新线程，循环处理任务
        let thread = thread::spawn(move || loop {
            // 加锁并接收任务，unwrap 处理锁和接收错误
            let message = receiver.lock().unwrap().recv();

            // 匹配接收结果
            match message {
                Ok(job) => {
                    // 收到任务，执行闭包
                    println!("Worker {id} got a job; executing.");
                    job();
                }
                Err(_) => {
                    // 通道关闭，退出循环
                    println!("Worker {id} disconnected; shutting down.");
                    break;
                }
            }
        });

        Worker {
            id,              // 保存工作者ID
            thread: Some(thread), // 保存线程句柄
        }
    }
}

webserver/src/main.rs

// 使用自定义库中的 ThreadPool
use web_server::ThreadPool;
use std::{
    fs,                            // 文件系统操作
    io::{prelude::*, BufReader},    // 输入输出操作
    net::{TcpListener, TcpStream},  // 网络操作
    thread,                         // 线程操作
    time::Duration,                 // 时间延迟
};

// 主函数，程序入口
fn main() {
    // 绑定到本地地址 127.0.0.1:7878，创建 TCP 监听器
    let listener = TcpListener::bind("127.0.0.1:7878").unwrap();

    // 创建包含 4 个线程的线程池
    let pool = ThreadPool::new(4);

    // 循环处理传入连接，限制为前 2 个请求（测试用）
    for stream in listener.incoming().take(2) {
        // 获取 TCP 流，unwrap 处理连接错误
        let stream = stream.unwrap();

        // 将处理任务提交给线程池
        pool.execute(|| {
            handle_connection(stream); // 在线程中处理连接
        });
    }

    // 主线程结束，打印关闭信息
    println!("Shutting down.");
}

// 处理客户端请求并发送响应
// 参数 mut stream: TcpStream 是可变的 TCP 流
fn handle_connection(mut stream: TcpStream) {
    // 创建缓冲读取器，读取请求数据
    let buf_reader = BufReader::new(&stream);

    // 读取请求第一行，unwrap 处理读取错误
    let request_line = buf_reader.lines().next().unwrap().unwrap();

    // 根据请求路径匹配状态和文件名
    let (status_line, filename) = match &request_line[..] {
        "GET / HTTP/1.1" => ("HTTP/1.1 200 OK", "hello.html"), // 根路径
        "GET /sleep HTTP/1.1" => {
            // 模拟延迟，睡眠 5 秒
            thread::sleep(Duration::from_secs(5));
            ("HTTP/1.1 200 OK", "hello.html") // 延迟后返回 hello.html
        }
        _ => ("HTTP/1.1 404 NOT FOUND", "404.html"), // 其他路径返回 404
    };

    // 读取文件内容，unwrap 处理读取错误
    let contents = fs::read_to_string(filename).unwrap();

    // 计算内容长度
    let length = contents.len();

    // 构造 HTTP 响应
    let response = format!("{status_line}\r\nContent-Length: {length}\r\n\r\n{contents}");

    // 发送响应，unwrap 处理写入错误
    stream.write_all(response.as_bytes()).unwrap();
}

运行流程

  1. 创建线程池：ThreadPool::new(4) 创建4个工作线程。
  2. 监听连接：listener.incoming().take(2) 限制2个请求（测试用）。
  3. 任务分发：使用 ThreadPool::execute() 将任务发送至线程池。
  4. 处理请求：
    • 解析HTTP请求，匹配 / 和 /sleep。
    • /sleep 路径模拟5秒延迟。
  5. 清理资源：Drop 实现优雅关闭线程池。

重要实现细节

  • 多线程安全：
    • Arc<Mutex<mpsc::Receiver<Job>>> 共享任务队列。
    • Mutex 确保安全访问。
  • 任务分发：
    • mpsc::channel() 创建任务通道。
    • send(job) 发送任务到工作线程。
  • 线程管理：
    • Worker 结构体管理线程生命周期。
    • Drop 释放资源并确保线程退出。

3.对比与总结

单线程 vs 多线程

  • 单线程：
    • 优点：简单，易于调试。
    • 缺点：阻塞式处理，性能低。
  • 多线程：
    • 优点：并发处理，高吞吐量。
    • 缺点：复杂度增加，需线程安全管理。

Rust特性

  • 所有权：TcpStream转移给处理函数，确保资源管理。
  • 借用：&stream 在 BufReader 中避免移动。
  • 并发：Arc 和 Mutex 实现线程安全共享。

重要实现细节

  • 错误处理：unwrap 简化代码，生产环境需 match。
  • 性能优化：BufReader 提高读取效率，Arc减少复制。

本实现展示了Rust在网络编程中的强大能力，结合所有权和并发特性，构建高效安全的Web服务器。未来可扩展支持更多路由、错误处理和HTTPS。
