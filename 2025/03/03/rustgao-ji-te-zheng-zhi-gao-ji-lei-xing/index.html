Rust高级特征之高级类型

Rust 的类型系统有一些我们曾经提到但没有讨论过的功能。首先我们从一个关于为什么 newtype 与类型一样有用的更宽泛的讨论开始。接着会转向类型别名（type aliases），一个类似于 newtype 但有着稍微不同的语义的功能。我们还会讨论 ! 类型和动态大小类型。

1. newtype 模式

1.1 介绍

newtype 模式用于在外部类型上实现外部 trait。由于 孤儿规则 (orphan rule) 限制，我们只能在当前 crate 本地定义的类型上实现 trait，或者在当前 crate 本地定义的 trait 上实现外部类型。

但 newtype 模式提供了一种解决方案，即：

  • 创建一个封装外部类型的新结构体（通常是元组结构体）。
  • 在这个新类型上实现所需的 trait。

1.2 示例：在 Vec<String> 上实现 Display

问题：

  • Display 是标准库 std::fmt 中的 trait。
  • Vec<T> 也是标准库类型。
  • 孤儿规则 不允许我们直接为 Vec<T> 实现 Display。

解决方案：

  • 创建一个 Wrapper 结构体，它内部存储 Vec<String>。
  • 在 Wrapper 上实现 Display，并代理 Vec<String> 的功能。
// 引入标准库中的 fmt 模块，用于格式化相关的 trait
use std::fmt;

// 定义一个名为 Wrapper 的元组结构体，包装一个 Vec<String>
struct Wrapper(Vec<String>);

// 为 Wrapper 类型实现 Display trait，用于自定义格式化输出
impl fmt::Display for Wrapper {
    // fmt 方法定义了如何将 Wrapper 格式化为字符串
    // &self 是当前实例的引用，f 是格式化器
    fn fmt(&self, f: &mut fmt::Formatter) -> fmt::Result {
        // 使用 write! 宏将 Vec<String> 的内容连接并格式化为 "[元素1,元素2,...]" 的形式
        // self.0 访问元组结构体的第一个字段（Vec<String>）
        // join(",") 将向量中的字符串用逗号连接
        write!(f, "[{}]", self.0.join(","))
    }
}

// 主函数，程序的入口
fn main() {
    let w = Wrapper(vec![String::from("Hello"), String::from("Wrapper")]);

    // {w} 调用 Display trait 的实现
    println!("w = {w}"); //w =[Hello,Wrapper]
}

解析：

  • Wrapper 结构体只是 Vec<String> 的简单封装。
  • 在 fmt::Display 实现中，self.0.join(", ") 访问 Vec<String> 的值，并用 , 连接字符串。
  • write! 宏用于格式化输出。

1.3 newtype 模式的优缺点

✅ 优点：

  • 可以为外部类型实现外部 trait，绕开孤儿规则。
  • 零运行时开销，Wrapper 在编译时会被优化掉。
  • 可提供额外的类型安全（比如封装 u64 作为 UserId，防止误用）。

⚠️ 缺点：

  • Wrapper 结构体默认不会继承 Vec<T> 的方法。
  • 必须手动实现 Vec<T> 的所有方法，或者使用 Deref 进行透明代理（见下文）。

2. Deref trait 代理方法

2.1 Deref 的作用

如果 Wrapper 需要继承 Vec<T> 的所有方法，可以实现 Deref，让 Wrapper 自动解引用为 Vec<T>。

2.2 示例：实现 Deref 代理 Vec<T>

use std::ops::Deref;

struct Wrapper(Vec<String>);

impl Deref for Wrapper {
    type Target = Vec<String>; // 让 Wrapper 解引用为 Vec<String>

    fn deref(&self) -> &Self::Target {
        &self.0 // 返回内部 Vec<String> 的引用
    }
}

fn main() {
    let w = Wrapper(vec![String::from("hello"), String::from("world")]);

    // 由于实现了 Deref，Wrapper 可以直接调用 Vec<String> 的方法
    println!("First element: {}", w[0]); // 访问 Vec<String> 的第一个元素
}

解析：

  • impl Deref for Wrapper 让 Wrapper 像 Vec<String> 一样使用。
  • type Target = Vec<String> 指定解引用的目标类型。
  • deref 方法返回 &self.0，这样 Wrapper 自动解引用为 Vec<String>。
  • 之后，Wrapper 可以直接使用 Vec<T> 的方法，如 w[0]。

3. 类型别名（Type Alias）

3.1 作用

  • 给已有类型定义新的名字，提升可读性。
  • 简化复杂类型，特别是涉及泛型时。

3.2 示例：简化 Box<dyn Fn(i32) -> i32>

// 定义类型别名 Thunk，表示一个动态分发的闭包 trait 对象
type Thunk = Box<dyn Fn(i32) -> i32>;

// Thunk 是类型别名，代表一个闭包 trait 对象
fn takes_long_type(f: Thunk) {
    // 调用闭包 f，传入参数 10，并打印结果
    println!("Result: {}", f(10));
}


// 该函数构造并返回一个闭包，作为 Thunk 类型返回
fn returns_long_type() -> Thunk {
    Box::new(|x| x + 1) 
}

fn main() {
    let f: Thunk = Box::new(|x| x * 2);

    // 调用 takes_long_type 函数，传入闭包 f
    // f 在函数中被调用，计算 10 * 2 并打印结果
    takes_long_type(f);
}

解析：

  • type Thunk = Box<dyn Fn(i32) -> i32>; 定义别名，简化闭包类型。
  • takes_long_type(f: Thunk) 直接使用 Thunk，提升可读性。
  • returns_long_type() 返回 Thunk，避免 Box<dyn Fn(i32) -> i32> 过长。

✅ 优势：

  • 提高代码可读性，减少冗长类型。
  • 避免多次重复 Box<dyn Fn(i32) -> i32>。

⚠️ 注意：

  • 类型别名不会创建新的类型，Thunk 仍然是 Box<dyn Fn(i32) -> i32>。

4. ! 类型（Never Type）

Rust 有一个特殊的类型，名为 !，在类型理论术语中被称为空类型（empty type），因为它没有任何值。我们更喜欢称它为never type，用于 panic!()、loop {},它在函数永不返回时充当返回类型的占位符。以下是一个例子：

fn never_returns() -> ! {
    panic!("This function will never return!");
}

这段代码的含义是：函数 bar 永不返回。返回 never 类型的函数被称为发散函数（diverging functions）。我们无法创建类型 ! 的值，因此 bar 永远不可能返回。

  • fn never_returns() -> ! 表示该函数不会返回。
  • panic!() 直接导致程序终止。

应用场景：

  • 死循环：loop {}。
  • 错误处理：panic!()、unreachable!()。

5. 动态大小类型和 Sized trait

Rust 需要知道有关类型的某些细节，例如为特定类型的值分配多少空间。这便是 Rust 类型系统中的一个特殊概念：动态大小类型（Dynamically Sized Types，DST）。这种类型也有时被称为 unsized types，它允许我们处理只有在运行时才知道大小的类型。

1.动态大小类型

str 就是一个典型的动态大小类型（DST）。在编译时，Rust 并不知道字符串的长度，只有在运行时才能知道。因此，不能像 let s1: str = "Hello there!" 这样声明一个变量，因为编译时无法确定 str 的大小。

考虑以下代码，它会导致编译错误：

let s1: str = "Hello there!"; // 编译错误，str 类型的大小在编译时无法确定
let s2: str = "How's it going?"; // 编译错误，str 类型的大小在编译时无法确定

这是因为 str 是动态大小的类型，直到运行时才知道它的实际大小。因此，我们不能创建 str 类型的变量，也不能将其作为函数参数。

2.&str 类型的解决方案

为了解决这个问题，我们使用 &str 类型。

&str 是一个引用类型，它存储着字符串的地址以及它的长度。由于 &str 包含了两个部分：一个是指向字符串数据的指针（地址），另一个是 usize 类型的长度信息，这样它的大小在编译时就变得可知了。&str 的大小始终是两个 usize 的大小，即固定的，这样就能保证编译器在编译时知道它的大小。

因此，代码可以改为以下形式：

let s1: &str = "Hello there!";  // 正确，&str 引用类型的大小在编译时已知
let s2: &str = "How's it going?";  // 正确，&str 引用类型的大小在编译时已知

3.动态大小类型的黄金规则

动态大小类型的黄金规则：必须将动态大小类型的值放置在某种指针后面。也就是说，不能直接使用 str 类型变量，而应使用 &str 类型，这样才会有一个明确的指针和长度信息。

Sized trait

为了处理动态大小类型（DST），Rust 提供了 Sized trait 来确定一个类型的大小是否在编译时是已知的。Sized trait 自动为编译器在编译时已知大小的类型实现。实际上，Rust 对每个泛型函数隐式地添加了 Sized 限定符，这意味着泛型函数默认只允许大小已知的类型。

考虑以下泛型函数定义：

fn generic<T>(t: T) {
    // 泛型函数，不能使用动态大小类型（DST）
}

Rust 会将其视为以下形式：

fn generic<T: Sized>(t: T) {
    // 泛型函数，必须是已知大小的类型
}

这意味着 generic 函数只能接收编译时已知大小的类型。然而，您可以通过 ?Sized 来放宽这一限制，使函数能够处理动态大小类型（DST）。

4.放宽 Sized 限制

如果我们希望泛型函数能够接受动态大小类型（DST），可以使用如下语法来放宽 Sized 的默认限制：

fn generic<T: ?Sized>(t: &T) {
    // 允许 t 是动态大小类型（DST）
}

在这个例子中，T: ?Sized 表示 T 可能是动态大小类型（DST），也可能是一个常规类型。?Sized 会覆盖泛型类型在编译时必须具有固定大小的默认规则。需要注意的是，?Sized 只能用于 Sized trait，而不能用于任何其他的 trait。

5.示例代码

以下是一个完整的示例，演示了如何使用 ?Sized 和动态大小类型（DST）：

// 一个使用动态大小类型的结构体
struct MyStruct {
    name: String,
}

// 定义一个泛型函数，该函数可以接受动态大小类型（DST）
fn print_name<T: ?Sized>(t: &T) {
    // 我们知道 t 是一个指向动态大小类型（DST）的引用
    println!("Name: {:?}", t);
}

fn main() {
    let name = "Alice".to_string();
    let my_struct = MyStruct { name };

    // 传递引用，允许传递动态大小类型（DST）
    print_name(&my_struct);
}

总结

  • 动态大小类型（DST）：类型的大小在编译时不可知，只能在运行时确定。例如，str 就是一个 DST 类型。
  • &str：是 str 类型的引用，可以在编译时确定其大小。
  • Sized trait：用于确定类型的大小是否在编译时已知。默认情况下，泛型函数只能用于已知大小的类型。
  • ?Sized：用于放宽 Sized 限定，允许接受动态大小类型（DST）。
